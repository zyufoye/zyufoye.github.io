Java反序列化漏洞初探

Created2023-06-29Updated2023-06-29

要了解Java反序列化漏洞，首先要知道什么是序列化和反序列化，其次要了解为什么序列化？在Java中有两个类与反序列化漏洞息息相关，是ObjectOutputStream类与ObjectInputStream类。该文章简述了反序列化漏洞原理，其中包含readObject方法和Java类中serialVersionUID的作用。

1.什么是序列化和反序列化？

Java序列化是指把Java对象转换为字节序列的过程；
Java反序列化是指把字节序列恢复为Java对象的过程；

2.为什么要序列化？

对象不止存储在内存中，它还需要在网络上传输，并且保存起来之后需要下次再加载出来，这时就需要序列化技术。
Java的序列化技术就是把对象转换成一串由二进制字节组成的数组，然后将这二进制数据保存在磁盘中，或在网络中传输。而后需要用到这对象时，磁盘或者网络接收者可以通过反序列化得到此对象，达到对象持久化的目的。

3. ObjectOutputStream类与ObjectInputStream类

3.1 ObjectOutputStream类

java.io.ObjectOutputStream 类，将Java对象的原始数据类型写出到文件,实现对象的持久存储。
一个对象想要序列化，必须满足两个条件：

  • 该类必须实现 java.io.Serializable 接口，Serializable是一个标记接口，不实现此接口的类将不会使任何状态序列化或反序列化，会抛出 NotSerializableException 。
  • 该类的所有属性必须是可序列化的。如果有一个属性不需要可序列化的，则该属性必须注明是瞬态的，使用transient关键字修饰。
